# 需求文档

## 一、应用概述

- **应用名称**：施工企业市场经营管理小程序
- **应用描述**：服务于施工企业市场经营条线，聚焦经营工作全流程线上化、规范化、数据化管理，覆盖企业经营层、一线经营人员、内勤资料员等全角色，实现从客户对接、项目跟踪、投标管控、成果归档、决策分析的全业务场景闭环管理。支持管理员灵活分配用户至小组，提供多维度可视化分析报表，并涵盖周报审阅批注与历史追溯、周报与任务双向关联、工作汇报任务指派概览、首页小组合同额看板，以及专为高层管理人员定制的领导专属数据报表系统等功能。投标管理模块支持完整的CRUD操作与详情页展示；知识库模块集成Supabase Storage实现文档全生命周期管理；系统设置模块提供完整的角色权限编辑能力；全系统各模块图标与文字支持按角色进行可见性控制。权限管理体系采用「用户—角色—权限」三层模型，以角色为权限分配中心，用户通过关联一个或多个角色继承权限，角色权限变更实时生效于所有关联用户。新增权限预览面板、权限对比工具、权限模板库与权限继承关系可视化图谱，全面提升权限管理的易用性与管理效率。

## 二、用户与使用场景

### 2.1 目标用户

共设置5类核心角色，所有账号仅可由超级管理员添加手机号后开通，不开放自主注册。系统采用「用户—角色—权限」三层模型，一个用户可关联一个或多个角色，用户最终权限为其所有关联角色权限的并集：

| 角色 | 核心定位 |
|------|----------|
| 超级管理员 | 系统最高权限管控、全模块配置管理 |
| 公司领导 | 查看决策、目标管控、任务督办、专属报表分析 |
| 经营中心人员（主要领导、分管领导、总经理助理、高级经理、一级职员、二级职员、三级职员） | 一线填报、业务执行、任务落地 |
| 资料员 | 内勤归档、数据补录、资料维护 |
| 系统管理员 | 后台配置、权限维护、系统管理 |

### 2.2 核心使用场景

- 经营人员移动端填报项目跟进记录、客户跟进记录，替代线下纸质/Excel
- 领导查看经营数据看板、指派任务、督办进度、查看年度经营目标完成情况
- 领导通过专属仪表盘实时监控全局KPI，进行深度钻取分析与定制报告生成
- 资料员归档投标信息、维护知识库文档，对投标记录进行完整的增删改查操作
- 管理员配置账号权限、模板字段，并对用户进行小组分配与管理
- 管理员通过角色编辑页面修改角色名称、显示名称及权限项配置，支持角色继承关系设置
- 管理员在用户管理界面为用户分配一个或多个角色，查看用户当前所属角色及权限概要
- 管理员使用权限预览面板实时查看任意用户或角色的完整权限总览
- 管理员使用权限对比工具对两个用户、两个角色或用户与角色之间进行权限差异比对
- 管理员通过权限模板库快速创建或批量修改用户/角色的权限配置
- 管理员通过权限继承关系可视化图谱直观了解用户、角色之间的权限继承与关联关系
- 全员接收任务指派、逾期提醒等消息通知
- 用户在个人界面查看自己所属的小组列表
- 领导与管理员查看小组目标完成趋势图表与对比分析报表
- 审阅人对周报进行批注并查看历史审阅记录
- 用户通过周报与任务的双向关联快速导航
- 用户在工作汇报工作台查看指派给我的任务与我指派的任务
- 具备权限的用户在首页查看并编辑小组合同额看板数据
- 管理员在系统设置中控制各模块图标与文字对不同角色的可见性

## 三、权限体系

### 3.1 权限模型说明

系统采用「用户—角色—权限」三层RBAC模型：

- **用户实体**：代表系统中的具体使用者，存储用户名、登录凭证等身份信息，一个用户可关联一个或多个角色（多对多关系）
- **角色实体**：代表一组预设的权限集合，用于定义职能或岗位，一个角色可包含一个或多个权限（多对多关系），支持角色继承关系配置
- **权限实体**：代表系统内可执行的最小操作或可访问的资源单位
- **权限计算规则**：用户的最终权限为其所有关联角色所拥有权限的并集
- **实时生效**：修改某角色权限后，变更自动实时生效于所有关联该角色的用户，无需重新登录
- **审计日志**：角色权限的修改历史及影响范围记录于系统日志，支持查阅

### 4.2 登录页

- 支持微信授权登录与账号密码登录两种方式
- 账号仅由超级管理员预先录入手机号后开通，不支持自主注册
- 登录成功后根据角色跳转至对应首页视图

### 4.3 首页（数据看板）

首页分为三个主要区域：

**上方区域：年度经营目标**
- 展示当年度整体经营目标及完成情况，包含目标值、已完成值、完成率等核心指标
- 数值由超级管理员或系统管理员在「年度经营目标配置页」录入与修改
- 公司领导、超级管理员、系统管理员可查看；经营中心人员与资料员不可见

**中间区域：小组年度目标**
- 根据「小组设置」中已创建的小组数量，动态展示对应数量的小组目标卡片
- 每个小组卡片展示：小组名称、小组年度目标值、已完成值、完成率
- 领导视图自动显示所有已设置小组的目标概览
- 小组数量由管理员在「小组设置页」动态配置，首页随之自动更新

**下方区域：小组合同额看板**
- 展示各小组合同额完成情况，详见 4.14 节

### 4.4 项目管理模块

**项目列表页**
- 展示当前用户权限范围内的项目列表，支持按项目名称、状态、负责人筛选
- 领导与管理员可查看全部项目；经营中心人员仅查看本人负责项目

**项目详情页**
- 展示项目基本信息、当前状态、负责人、跟进记录等
- 支持编辑项目基本信息（权限内用户）

**项目内容填写页**
- 支持自定义项目模板，管理员可在「项目内容模板配置页」选择或创建模板
- 无论使用哪种模板，均必须具备以下核心功能：
  - 选择具体项目后，可填写对应的内容说明
  - 支持「单选项目单内容」模式：选择一个项目，填写一条独立内容
  - 支持「多选项目多内容」模式：同时选择多个项目，分别为每个项目填写独立内容
  - 内容填写界面简洁直观，支持文本输入与附件上传（图片、文件）
- 填写完成后提交保存，内容与对应项目绑定存储

### 4.5 客户管理模块

**客户列表页**
- 展示客户基本信息列表，支持按客户名称、行业、负责人筛选
- 领导与管理员可查看全部客户；经营中心人员仅查看本人负责客户

**客户详情/跟进记录页**
- 展示客户详细信息及历史跟进记录
- 支持新增跟进记录，填写跟进时间、跟进方式、跟进内容、下次跟进计划
- 支持附件上传

### 4.6 任务管理模块

**任务列表页**
- 展示本人指派出去的任务与被指派的任务，支持按状态、截止日期筛选
- 领导/管理员可查看全部任务

**任务详情页**
- 展示任务标题、描述、指派人、执行人、截止日期、当前状态、进度记录
- 执行人可更新任务进度与状态
- 新增「关联周报」板块：
  - 列表展示所有提及此任务的周报记录
  - 每条记录展示：周报所属周期、汇报人姓名、跳转至对应周报详情页的链接
  - 若无关联周报，展示「暂无关联周报」提示

**任务指派页（领导/管理员）**
- 填写任务标题、描述、指派对象（支持多选）、截止日期、优先级
- 提交后系统向被指派人发送消息通知
- 支持逾期自动提醒

### 4.7 投标管理模块

**投标信息列表页**
- 展示已登记的投标与中标信息，支持按项目名称、状态、时间筛选
- 所有角色均可查看；编辑与删除操作仅资料员、超级管理员、系统管理员可见

**投标/中标登记页（新增/编辑）**
- 填写字段：项目名称、投标日期、投标金额、投标结果（中标/未中标）、中标金额（中标时填写）、中标日期（中标时填写）、备注说明
- 支持附件上传（标书、中标通知书等）
- 新增与编辑共用同一页面，编辑时原有信息预填充

**投标详情页**
- 展示完整投标信息，包含以下字段：
  - 项目名称
  - 投标日期
  - 投标金额
  - 中标金额
  - 中标日期
  - 备注说明
  - 附件列表（每条附件支持在线预览与下载）
- 编辑与删除按钮仅对资料员、超级管理员、系统管理员角色可见
- 删除操作需弹出确认对话框，用户确认后方可执行，防止误删
- 编辑操作跳转至「投标/中标登记页」并预填充当前数据
- 数据操作前进行角色权限校验，非授权角色无法触发编辑/删除接口

### 4.8 知识库模块

**知识库文档列表页**
- 展示已上传的知识库文档，支持按分类、关键词搜索
- 支持按文档名称、分类、上传时间、文件大小进行搜索与排序
- 所有角色均可在权限范围内预览与下载
- 资料员、超级管理员、系统管理员可上传、编辑、删除文档
- 每条文档记录展示：文档名称、文档分类、文件大小、上传人、上传时间

**文档上传功能**
- 支持上传格式：PDF、Word（.doc/.docx）、Excel（.xls/.xlsx）
- 单文件大小限制：10MB，超出时提示「文件大小不能超过10MB」
- 上传时填写/选择：文档名称、文档分类
- 文件上传至Supabase Storage，系统在documents表中存储以下元数据：
  - 文档名称
  - 文档分类
  - 文件大小
  - 上传人
  - 上传时间
  - 文件在Supabase Storage中的路径

**文档预览/下载页**
- 在线预览：支持PDF格式在线预览

**权限预览面板页**

  - 以分组列表形式展示该对象的全部权限项，分组维度包括：菜单访问权限、操作按钮权限、数据范围权限

  - 提供权限来源筛选控件（直接拥有/角色继承/继承链路），支持单选或多选
  - 筛选条件变更后列表实时刷新，无需重新加载页面
- **快捷操作**：
  - 面板内提供「发起对比」按钮，点击后以当前对象为基准，跳转至权限对比工具页并预填充一侧数据
  - 若当前对象为角色，提供「前往编辑」快捷入口，跳转至对应角色编辑页
- **加载状态**：数据加载过程中展示加载动画，加载失败时展示「权限信息加载失败，请刷新重试」提示

**权限对比工具页**

- **页面定位**：支持在两个用户、两个角色或用户与角色之间进行权限配置的差异比对，帮助管理员快速识别权限差异
- **对比对象选择区域**：
  - 页面顶部提供左右两侧对象选择区域，每侧独立支持选择「用户」或「角色」类型
  - 每侧提供搜索输入框，支持按姓名/角色名称快速定位
  - 两侧对象均选定后，点击「开始对比」按钮触发对比计算，对比结果实时渲染
- **对比结果展示区域**：
  - 以三列布局展示对比结果：左侧对象独有权限、双方共有权限（交集）、右侧对象独有权限
  - 独有权限项以高亮颜色标注（左侧独有与右侧独有使用不同颜色），共有权限项以中性色展示
  - 支持按权限类型（菜单访问/操作按钮/数据范围）筛选对比结果
  - 页面顶部展示对比摘要：左侧独有权限数量、共有权限数量、右侧独有权限数量
- **交互操作**：
  - 支持在对比结果页直接点击任意权限项，查看该权限项的详细说明
  - 提供「重新选择」按钮，清空当前对比结果并重新选择对比对象
- **引导提示**：页面提供简短操作说明，引导管理员完成对比流程，降低学习成本

**权限模板库页**

- **页面定位**：建立可复用的权限配置模板库，支持基于模板快速创建或批量修改用户/角色的权限配置
- **模板列表区域**：
  - 以卡片或列表形式展示所有已创建的权限模板
  - 每条模板记录展示：模板名称、模板描述、包含权限数量、创建人、最后修改时间、版本号
  - 支持按模板名称搜索
  - 每条模板记录提供操作入口：编辑、删除、应用
- **模板编辑页**：
  - 支持填写/修改模板名称与模板描述
  - 通过勾选方式配置该模板包含的权限项，权限项列表与系统权限体系保持一致
  - 支持保存为新版本：每次修改提交时，系统自动记录版本号与修改时间，保留历史版本列表供查阅
  - 提供「删除模板」功能，点击后弹出确认对话框，用户确认后方可执行
- **应用模板功能**：
  - 在模板列表页点击「应用」后，弹出应用目标选择弹窗
  - 支持选择应用目标类型：单个用户、单个角色、批量用户、批量角色
  - 选择目标后展示应用预览：当前目标权限与模板权限的差异摘要（新增权限数量、移除权限数量）
  - 确认应用前弹出二次确认提示：「应用模板将覆盖所选目标的现有权限配置，是否确认？」，用户确认后执行
  - 应用完成后展示成功提示，并提供「查看权限详情」快捷入口
- **版本管理**：
  - 每个模板维护独立的版本历史列表，展示：版本号、修改时间、修改人、变更摘要
  - 支持查看历史版本的权限配置详情（只读）
  - 不支持版本回滚（本期仅支持查阅历史版本）

**权限继承关系图谱页**

- **页面定位**：以图形化关系图谱直观展示用户、角色之间的权限继承与关联关系，支持节点交互操作
- **图谱展示区域**：
  - 以有向图形式渲染权限继承关系，节点代表用户或角色，有向边代表继承或关联关系
  - 节点类型以不同图标或颜色区分：用户节点、角色节点
  - 有向边方向表示权限流转方向（从父角色指向子角色，从角色指向用户）
  - 支持多层级继承路径的清晰呈现，层级关系通过纵向或横向布局展示
  - 图谱支持缩放（放大/缩小）与拖拽平移，适应不同规模的关系网络
- **节点交互**：
  - 点击任意节点，在图谱右侧或底部弹出该节点的权限详情面板，展示其完整权限列表（含来源标注）
  - 权限详情面板提供「前往编辑」快捷入口：用户节点跳转至用户编辑页，角色节点跳转至角色编辑页
  - 悬停节点时高亮显示与该节点直接相关的所有连接边，便于追踪继承路径
- **筛选与聚焦**：
  - 提供搜索输入框，支持按用户姓名或角色名称快速定位目标节点，定位后图谱自动聚焦并高亮该节点
  - 提供节点类型筛选控件（仅显示用户/仅显示角色/全部显示），减少图谱复杂度
- **性能处理**：
  - 图谱初始加载时展示加载动画，加载完成后渲染图形
  - 节点数量较多时（超过系统配置阈值），默认仅展示角色层级关系图，用户节点按需展开，避免图谱过于密集
  - 加载失败时展示「关系图谱加载失败，请刷新重试」提示
- **操作引导**：页面提供简短操作说明（如「点击节点查看权限详情」「滚轮缩放图谱」），帮助管理员快速上手

**小组设置页（含成员管理）**

小组列表区域：
- 展示所有已创建小组，每行显示：小组名称、年度目标值、当前成员数量、操作入口
- 支持创建、编辑、删除小组
- 每个小组可配置：小组名称、小组年度目标值

小组成员管理区域：
- 点击任意小组进入成员管理详情页，展示该小组当前全部成员列表（姓名、角色、加入时间）
- **单个分配**：管理员可从系统用户列表中选择单个用户，将其分配至当前小组
- **批量分配**：
  - 管理员可勾选多名用户，一次性将其分配至一个或多个小组
  - 操作路径：选择用户 → 选择目标小组（支持多选）→ 确认分配
- **单个移除**：管理员可将单个成员从当前小组移除
- **批量移除**：
  - 管理员可在成员列表中勾选多名成员，一次性将其从当前小组移除
  - 支持跨小组批量移除：选择多名用户后，可选择从一个或多个小组中同时移除
- 所有分配与移除操作实时生效，首页小组目标卡片、个人中心「我的小组」页面同步更新
- 操作完成后系统向被分配/移除的用户推送消息通知

**项目内容模板配置页**
- 超级管理员与系统管理员可创建、编辑、删除项目内容模板
- 每个模板可配置：模板名称、模板字段（文本、附件等类型）
- 支持将模板设为默认模板，项目内容填写时可选择使用

**年度经营目标配置页**
- 超级管理员与系统管理员可录入、修改当年度整体经营目标值及各小组目标值

**模块可见性配置页**

访问权限：超级管理员、系统管理员

- 以模块/功能项列表形式展示系统内所有模块、子模块及其对应的导航图标与文字标签
- 每一项模块/功能项均可按角色维度独立配置可见性：
  - 配置维度：超级管理员、公司领导、经营中心人员、资料员、系统管理员
  - 配置方式：勾选/取消勾选对应角色，控制该模块图标与文字对该角色是否可见
- 配置粒度覆盖：底部导航栏入口、各模块内子页面入口、功能操作按钮（如编辑、删除等）
- 配置提交后实时生效，对应角色用户刷新后立即看到更新后的界面
- 超级管理员自身的可见性配置不可被修改，始终保持全量可见

**系统日志页**
- 超级管理员与系统管理员可查看系统操作日志，包括账号操作、数据修改、权限变更（含角色权限修改历史及影响用户范围）、小组成员变更、权限模板应用记录等

## 五、业务规则与逻辑

### 5.1 账号与登录规则

- 所有账号由超级管理员预先录入手机号后开通，不支持自主注册
- 同一手机号仅可绑定一个账号
- 账号被停用或锁定后立即失效，无法登录
- 登录成功后根据角色加载对应权限视图

### 5.2 首页动态渲染规则

- 首页下方小组目标区域的卡片数量与「小组设置」中已创建的小组数量严格对应
- 管理员新增或删除小组后，首页小组目标区域实时更新，无需手动刷新
- 年度经营目标数值修改后，首页上方区域实时同步展示最新数值
- 小组合同额看板中「年度合同额目标」修改后实时同步至看板展示

### 5.3 小组成员管理规则

- 一个用户可同时属于多个小组，不限制所属小组数量
- 批量分配时，目标小组支持多选，系统将所选用户同时写入所有目标小组
- 批量移除时，支持选定多名用户并指定从一个或多个小组中移除，系统逐一解除对应关联关系
- 分配与移除操作提交后立即生效，个人中心「我的小组」页面、首页小组目标卡片同步更新
- 操作完成后系统向被操作用户推送消息通知（分配通知/移除通知）
- 删除小组时，该小组下所有成员关联关系同步清除，成员的其他小组归属不受影响
- 添加用户时在「小组选择」字段完成的分配，与小组设置页的成员分配逻辑一致，立即生效

### 5.4 小组目标统计分析规则

- 统计数据来源：各小组年度目标值（由管理员在年度经营目标配置页录入）与实际完成值（由业务数据汇总计算）
- 月度粒度：按自然月统计各小组当月完成值与目标值，计算当月完成率
- 季度粒度：按自然季度（Q1/Q2/Q3/Q4）汇总各小组完成值与目标值，计算季度完成率
- 年度粒度：按自然年汇总，展示全年累计完成情况
- 进度变化计算：当前周期完成率与上一同类周期完成率之差，正值为增长，负值为下降
- 分析报表摘要生成规则：取当前筛选时间范围内完成率最高的小组标注为「表现优异」，完成率最低且低于60%的小组标注为「需关注」；若所有小组完成率均高于60%，则不展示「需关注」标注
- 图表数据导出格式：支持导出为表格文件，包含各小组在所选时间周期内的全部关键指标数据

### 5.5 项目内容填写规则

- 进入项目内容填写页时，系统展示可用模板列表，用户选择模板后进入填写界面
- 单选项目单内容模式：用户选择一个项目，填写一条内容后提交
- 多选项目多内容模式：用户勾选多个项目，系统为每个项目生成独立填写区域，用户分别填写后统一提交
- 每条内容记录与对应项目绑定，可在项目详情页查看历史内容记录

### 5.6 任务流转规则

- 任务状态流转：待执行 → 进行中 → 已完成
- 任务指派后系统立即推送消息通知至被指派人
- 任务到达截止日期仍未完成时，系统自动推送逾期提醒至执行人与指派人
- 执行人可在任务详情页更新进度记录与状态

### 5.7 数据权限隔离规则

- 经营中心人员仅可查看、编辑本人负责的项目与客户数据
- 公司领导与超级管理员可查看全量经营数据
- 任务数据仅指派人、被指派人与管理员可见
- 小组目标统计分析页及领导专属报表模块仅超级管理员、公司领导、系统管理员可访问
- 周报审阅历史：汇报人本人可查看本人周报的审阅历史；审阅人及管理员可查看全部周报的审阅历史
- 领导专属报表模块中所有数据均基于当前用户管理权限自动过滤，不展示权限范围外的数据
- 投标详情页编辑与删除操作仅资料员、超级管理员、系统管理员可见，其他角色不展示操作入口，且后端接口同步进行角色权限校验
- 权限校验基于用户所有关联角色的权限并集进行实时计算，前端界面根据权限动态显示或隐藏对应功能按钮与菜单
- 权限管理增强模块（权限预览面板、权限对比工具、权限模板库、权限继承关系图谱）仅超级管理员与系统管理员可访问

### 5.8 账号安全规则

- 密码修改需验证原密码，验证通过后方可设置新密码
- 登录设备管理支持远程退出指定设备，退出后该设备上的登录态立即失效
- 登录历史记录保留近期登录信息，包括登录时间、设备标识、IP地址
- 账号锁定由管理员手动操作，锁定后账号无法登录，需管理员手动解锁

### 5.9 数据实时同步规则

- 所有数据更新（目标值修改、小组配置变更、成员分配/移除、任务状态更新等）提交后立即反映在对应界面，无需手动刷新
- 领导专属仪表盘KPI数据支持定时自动更新与手动刷新，定时更新频率由系统配置

### 5.10 周报审阅批注规则

- 审阅人在审阅页面执行「通过」或「需修改」操作时，可同步填写批注内容
- 批注内容支持格式化文本（加粗、项目符号），不强制要求必填，但建议填写以便追溯
- 每次审阅操作（含批注）自动保存并关联至对应周报，形成独立的审阅历史条目
- 审阅历史按时间倒序排列，每条记录包含：审阅人姓名、审阅时间、审阅操作、批注内容
- 审阅完成后系统向汇报人推送消息通知，通知内容包含审阅结果（通过/需修改）

### 5.11 周报与任务双向关联规则

- 周报填写时，用户可在「本周工作内容」或「下周计划」中手动关联已有任务
- 系统在周报详情页自动识别并提取已关联的任务，生成可点击的任务卡片
- 任务卡片展示：任务名称、任务状态、截止日期；点击后跳转至任务详情页
- 任务详情页「关联周报」板块自动汇总所有提及该任务的周报记录，展示：周报所属周期、汇报人、跳转链接
- 关联关系在周报提交时建立，周报删除时对应关联关系同步清除，任务本身不受影响

### 5.12 小组合同额看板规则

- 「年度合同额目标」由具备编辑权限的用户（超级管理员、系统管理员、小组长）手动录入与修改
- 「已完成合同额」通过与项目/财务数据接口自动同步，不允许手动修改
- 目标完成率计算公式：已完成合同额 ÷ 年度合同额目标 × 100%
- 年度合同额目标为0或未配置时，完成率展示「-」，不进行除零计算
- 编辑操作需经过确认步骤方可提交，防止误操作
- 数据同步异常时展示提示，不影响其他看板区域正常展示

### 5.13 领导专属报表模块规则

**仪表盘KPI数据规则**
- KPI数据来源于系统各业务模块的实时汇总计算，数据范围基于当前用户管理权限自动过滤
- 支持定时自动更新（更新频率由系统配置）与手动刷新两种方式
- 与上期对比的变化趋势计算：当前值与上一同类周期值之差，正值为增长，负值为下降

**深度钻取分析规则**
- 钻取操作仅在有下层明细数据时可用，无明细数据时钻取入口置灰并提示「暂无明细数据」
- 钻取路径最多支持三级，第三级为最细粒度明细记录，不再提供下钻入口
- 每级钻取视图的数据范围与上级筛选条件保持一致，不扩展权限范围外的数据

**智能预警规则**
- 预警阈值由系统预置，本期不支持用户自定义阈值配置
- 预警触发后系统同步向有权限用户推送消息通知（通知类型：智能预警通知）
- 同一预警条件在同一周期内仅触发一次通知，避免重复推送
- 预警标记为「已处理」后，若同一条件再次触发，系统重新生成新的预警条目

**定制报告生成规则**
- 用户至少需选择一个指标与一个维度方可生成预览，否则提示「请至少选择一个指标和一个维度」
- 报告预览数据与当前系统数据实时同步，每次调整选项后重新拉取最新数据渲染
- 导出操作基于当前预览内容生成PDF文件，导出内容与预览内容严格一致

### 5.14 投标信息CRUD规则

- 新增投标记录：资料员、超级管理员、系统管理员可创建新投标记录，填写完整字段后提交保存
- 查看投标详情：所有角色均可查看投标详情页，但编辑与删除操作入口仅对资料员、超级管理员、系统管理员可见
- 编辑投标记录：点击编辑后跳转至登记页并预填充原有数据，修改后提交覆盖原记录
- 删除投标记录：点击删除后弹出确认对话框，用户确认后执行删除，操作不可撤销
- 所有写操作（新增/编辑/删除）在后端接口层进行角色权限校验，非授权角色的请求直接拒绝

### 5.15 知识库文档管理规则

- 文档上传时系统校验文件格式（仅允许PDF、Word、Excel）与文件大小（不超过10MB），不符合条件时阻止上传并给出提示
- 文件上传至Supabase Storage后，系统在documents表中写入对应元数据记录（文档名称、文档分类、文件大小、上传人、上传时间、文件路径）
- 文档删除时同步删除Supabase Storage中的对应文件及documents表中的元数据记录
- 搜索功能基于文档名称与文档分类字段进行匹配查询
- 排序功能支持按上传时间（升序/降序）与文件大小（升序/降序）切换
- PDF格式文档支持在线预览，其他格式仅支持下载

### 5.16 角色权限变更实时生效规则

- 管理员在角色编辑页提交权限变更后，系统立即更新该角色的权限配置
- 已登录的该角色用户在下次发起请求时（无需重新登录）即可感知权限变化，受限操作入口自动隐藏或置灰
- 角色删除操作需满足前提条件：该角色下无任何用户；否则系统阻止删除并给出提示
- 角色权限变更（含继承关系调整）自动实时生效于所有关联该角色的用户，变更记录写入系统日志

### 5.17 模块可见性控制规则

- 模块可见性配置以角色为维度，每个模块/功能项对每个角色独立配置
- 配置提交后实时生效，对应角色用户界面中不可见的模块图标与文字标签不渲染，不仅隐藏而是不展示
- 超级管理员的可见性配置不可被修改，始终保持全量可见
- 可见性控制仅影响前端展示，不影响后端权限校验逻辑；即使某模块对某角色不可见，后端接口仍按权限体系进行访问控制

### 5.18 用户与角色联动规则

- 一个用户可同时关联一个或多个角色，用户最终权限为所有关联角色权限的并集
- 为用户新增或解除角色关联后，权限变更立即生效，无需重新登录
- 修改某角色的权限项或继承关系后，所有关联该角色的用户权限自动同步更新
- 删除角色前，系统检查该角色下是否存在关联用户：若存在，阻止删除并提示「该角色下存在用户，请先调整用户角色后再删除」
- 用户编辑界面的「当前权限概要」区域实时反映用户通过所有关联角色获得的权限并集，供管理员核查
- 角色继承关系调整后，子角色的有效权限（继承权限+自有权限）实时重新计算并生效于所有关联该子角色的用户
- 所有角色关联/解除操作及权限变更均记录于系统日志，包含操作人、操作时间、变更内容及影响用户范围

### 5.19 权限预览面板规则

- 权限预览面板展示的权限数据实时从系统权限计算引擎获取，确保与用户/角色当前实际权限一致
- 权限来源标注规则：直接赋予该对象的权限标注为「直接拥有」；通过关联角色获得的权限标注为「角色继承」并注明来源角色名称；通过角色继承链路从上级角色获得的权限标注为「继承链路」并注明完整继承路径
- 筛选条件变更后权限列表实时刷新，不重新请求接口，基于已加载数据进行前端过滤
- 数据加载失败时展示错误提示，不展示空列表

### 5.20 权限对比工具规则

- 对比计算基于两个对象各自的完整有效权限集合（含继承权限），而非仅直接赋予的权限
- 对比结果分类：左侧独有（仅左侧对象拥有）、共有（双方均拥有）、右侧独有（仅右侧对象拥有）
- 对比结果按权限类型分组展示，同一权限类型内按权限名称排序
- 两侧对象均需选定后方可触发对比，单侧未选定时「开始对比」按钮置灰

### 5.21 权限模板库规则

- 模板名称在系统内唯一，创建或编辑时若名称重复，提示「模板名称已存在，请使用其他名称」
- 应用模板时，目标对象的权限配置将被模板权限完全覆盖，原有权限配置不保留（覆盖前展示差异预览并要求二次确认）
- 批量应用模板时，系统逐一处理每个目标对象，全部处理完成后展示汇总结果（成功数量/失败数量）
- 模板版本历史仅支持查阅，不支持回滚（本期限制）
- 删除模板不影响已应用该模板的用户/角色的现有权限配置
- 权限模板的应用操作记录于系统日志，包含操作人、操作时间、应用模板名称、目标对象列表

### 5.22 权限继承关系图谱规则

- 图谱数据来源于系统当前的用户—角色关联关系与角色继承关系，实时反映最新配置
- 节点数量超过系统配置阈值时，默认仅展示角色层级关系，用户节点需手动展开（点击角色节点后展示关联用户列表）
- 图谱中的编辑操作（通过节点详情面板跳转至编辑页）完成后，返回图谱页时自动刷新图谱数据
- 图谱不支持直接在图谱视图内拖拽修改继承关系，所有修改须通过角色编辑页完成
