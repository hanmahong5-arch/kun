# 需求文档

## 一、应用概述

- **应用名称**：施工企业市场经营管理小程序
- **应用描述**：服务于施工企业市场经营条线，聚焦用户访问全流程审批管控与客户数据分析。系统底层采用标准RBAC角色权限体系，设超级管理员、审批管理员、普通用户三个核心角色，权限严格隔离。用户主表以手机号为唯一标识，账号状态固定为待审批/已通过/已驳回三态。所有审批操作写入独立全量审批日志表，不可删除、不可修改。本期核心目标：实现【用户访问必须管理员审批】完整闭环，仅已通过用户可进入主功能区；同时新增客户分级多选筛选、负责人筛选功能，支持负责人权限级别设置（主负责人/协助负责人）、客户转移功能，以及客户类型分析数据中心。

## 二、底层架构规范（固定，禁止修改）

### 2.1 数据表设计

#### 用户主表（users）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| user_id | 唯一标识 | 用户唯一ID，主键 |
| phone | 字符串，唯一 | 手机号，系统内唯一标识 |
| name | 字符串 | 姓名 |
| apply_reason | 字符串 | 申请理由 |
| account_status | 枚举 | 账号状态：待审批 / 已通过 / 已驳回（固定枚举，禁止自定义扩展） |
| apply_time | 时间戳 | 申请时间 |
| approve_time | 时间戳 | 审批时间（审批完成后写入） |
| approver_id | 外键 | 审批人user_id（审批完成后写入） |
| approve_remark | 字符串 | 审批备注 |

**账号状态枚举说明**：
- 待审批：用户已注册，等待审批管理员或超级管理员审核
- 已通过：审批通过，用户可正常进入主功能区
- 已驳回：审批驳回，用户无法进入任何业务功能区

**禁止事项**：
- 禁止在用户主表中新增任何枚举值之外的账号状态
- 禁止新增冗余字段
- 角色关联通过独立的用户角色关联表（user_roles）实现，用户主表不存储角色信息

#### 用户角色关联表（user_roles）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | 唯一标识 | 主键 |
| user_id | 外键 | 关联用户主表user_id |
| role | 枚举 | 角色：super_admin / approval_admin / normal_user |

#### 审批日志表（approval_logs）

独立全量审批日志表，所有审批操作均写入此表，**不可删除、不可修改任何记录**：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| approval_id | 唯一标识 | 审批记录唯一ID，主键 |
| approval_type | 枚举 | 审批类型：用户审批 / 功能审批 |
| target_id | 外键 | 审批目标ID（用户审批时为user_id，功能审批时为feature_id） |
| operator_id | 外键 | 操作人user_id |
| action | 枚举 | 操作动作：通过 / 驳回 |
| approve_remark | 字符串 | 审批备注 |
| operate_time | 时间戳 | 操作时间 |
| operate_ip | 字符串 | 操作IP |

**强制约束**：
- 审批日志表记录只可新增，禁止任何UPDATE或DELETE操作
- 后端接口层强制拦截对审批日志表的修改与删除请求

#### 客户表（customers）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| customer_id | 唯一标识 | 客户唯一ID，主键 |
| customer_name | 字符串 | 客户名称 |
| customer_type | 枚举 | 客户类型：央企 / 政府 / 省属 / 其他 |
| customer_level | 枚举 | 客户分级：新客户 / 老客户 |
| region | 字符串 | 所属地区 |

| interaction_count | 整数 | 互动次数 |
| login_count | 整数 | 登录频率（累计登录次数） |
| created_at | 时间戳 | 创建时间 |
| updated_at | 时间戳 | 更新时间 |

**枚举说明**：
- 客户类型：央企、政府、省属、其他（固定枚举，禁止自定义扩展）
- 客户分级：新客户、老客户（固定枚举，禁止自定义扩展）

#### 客户负责人关联表（customer_owners）

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | 唯一标识 | 主键 |
| customer_id | 外键 | 关联客户表customer_id |
| user_id | 外键 | 关联用户主表user_id |

**枚举说明**：
- 负责人类型：主负责人、协助负责人（固定枚举，禁止自定义扩展）

#### 客户转移记录表（customer_transfer_logs）

| transfer_time | 时间戳 | 转移时间 |
| operator_id | 外键 | 操作人user_id |

**强制约束**：
- 转移记录只可新增，禁止任何UPDATE或DELETE操作

### 2.2 RBAC角色权限体系（固定，禁止修改）

| 角色 | 标识 | 权限范围 |
|------|------|----------|
| 超级管理员 | super_admin | 系统全部权限：管理所有用户、执行用户审批、查看审批日志、修改系统配置、访问客户类型分析数据中心、执行客户转移操作 |
| 审批管理员 | approval_admin | 仅限：用户审批权限；可查看审批工作台、执行通过/驳回操作；无系统配置权限；无客户类型分析数据中心访问权限 |
| 普通用户 | normal_user | 仅限：账号审批通过后，进入主功能区；可使用客户分级多选筛选、客户类型多选筛选、负责人筛选功能；主负责人拥有完整编辑权限；协助负责人只能查看和添加跟进记录；无任何管理、审批、配置权限 |

**权限边界强制约束**：
- 审批管理员不可访问系统配置页，后端接口层强制拦截越权请求
- 普通用户不可访问任何管理后台页面，后端接口层强制拦截越权请求
- 客户类型分析数据中心仅超级管理员可访问，后端接口层强制拦截越权请求
- 客户编辑操作仅主负责人可执行，后端接口层强制拦截协助负责人的编辑请求
- 所有表之间通过唯一ID关联，禁止跨表直接引用非ID字段
- 所有状态字段严格遵循枚举值，后端接口层拒绝枚举值之外的状态写入请求

## 三、用户与使用场景

### 3.1 目标用户

| 角色 | 核心定位 |
|------|----------|
| 超级管理员 | 系统全权管理，负责用户审批、系统配置、客户数据分析、客户转移操作 |
| 审批管理员 | 专职审批，负责用户注册审批 |
| 普通用户 | 提交注册申请，审批通过后进入主功能区，使用客户筛选功能；主负责人可编辑客户，协助负责人可查看和添加跟进记录 |

### 3.2 核心使用场景

- 用户使用手机号提交注册申请，填写姓名与申请理由，等待审批
- 用户在申请进度页查看当前账号状态及审批结果
- 审批管理员或超级管理员在审批工作台查看待审批用户，执行通过或驳回操作，驳回必填原因
- 审批操作完成后自动通知用户，所有操作全量写入审批日志表
- 账号状态为已通过的用户登录后进入主功能区
- 待审批/已驳回用户完全无法进入主功能区
- 已通过用户在客户列表页通过多选客户分级、客户类型标签及负责人筛选组合筛选客户
- 主负责人在客户详情页对客户信息进行完整编辑；协助负责人只能查看和添加跟进记录
- 超级管理员执行客户转移，支持单个或批量转移，填写转移原因，转移历史可在客户详情页追溯
- 超级管理员在客户类型分析数据中心查看各维度客户数据图表，支持手动刷新与图表交互

## 四、页面结构与功能说明

### 4.1 整体页面结构

```
施工企业市场经营管理小程序
├── 注册申请页（pages/auth/register）
├── 申请进度查询页（pages/auth/status）
├── 主功能区占位页（pages/home/index）【仅已通过用户可访问】
├── 客户管理
│   ├── 客户列表页（pages/customer/index）【支持多选分级、类型、负责人筛选】
│   └── 客户详情页（pages/customer/detail）【含负责人信息、转移历史、跟进记录】
├── 审批工作台（仅超级管理员、审批管理员可见）
│   └── 用户审批列表页（pages/admin/user-approval/index）
│       └── 用户审批详情页（pages/admin/user-approval/detail）
└── 系统管理（仅超级管理员可见）
    ├── 用户管理页（pages/admin/user-manage/index）
    │   └── 用户详情页（pages/admin/user-manage/detail）
    ├── 审批日志页（pages/admin/approval-log/index）
    ├── 客户转移页（pages/admin/customer-transfer/index）
    └── 客户类型分析数据中心（pages/admin/customer-analysis/index）
```

### 4.2 注册申请页

页面路径：pages/auth/register

- 用户填写手机号（必填）、姓名（必填）、申请理由（选填），点击「提交申请」完成注册
- 提交后账号状态自动置为「待审批」，跳转至申请进度查询页
- 同一手机号已存在账号时，提示「该手机号已注册」，阻止重复创建账号
- 已注册用户（任意状态）可通过手机号直接进入申请进度查询页查看状态，无需重新提交
- 保留微信授权登录入口，微信授权后自动获取手机号：
  - 若手机号未注册，自动跳转至注册申请页并预填手机号
  - 若手机号已注册，直接进入申请进度查询页或主功能区（依账号状态判断）

### 4.3 申请进度查询页

页面路径：pages/auth/status

访问权限：所有已提交申请的用户（待审批、已通过、已驳回）

- 展示当前账号状态（待审批 / 已通过 / 已驳回）
- 展示用户提交的申请信息：手机号、姓名、申请理由、申请时间
- **待审批状态**：展示「您的账号正在审批中，请耐心等待」提示；提供「退出」入口
- **已通过状态**：展示「您的账号已通过审批」提示及审批时间；提供「进入应用」按钮，点击跳转至主功能区
- **已驳回状态**：展示「您的账号申请已被驳回」提示及驳回原因（来自approve_remark字段）；提供「重新申请」入口，用户可修改申请理由后重新提交，账号状态重置为「待审批」，申请时间更新；提供「退出」入口
- 页面进入时实时查询当前账号状态，确保展示最新审批结果

### 4.4 主功能区占位页

页面路径：pages/home/index

访问权限：仅账号状态为「已通过」的用户

- 本期作为主功能区入口占位页，展示「欢迎使用施工企业市场经营管理系统」提示
- 后续业务功能模块通过功能模块管理流程上线后在此展示
- 提供「个人信息」入口，展示当前用户手机号、姓名、角色、账号状态
- 提供「客户管理」快捷入口，跳转至客户列表页
- 超级管理员与审批管理员在此页额外展示「审批工作台」快捷入口
- 超级管理员在此页额外展示「系统管理」快捷入口（含客户类型分析数据中心入口、客户转移入口）
- 提供「退出登录」入口

### 4.5 客户列表页

页面路径：pages/customer/index

访问权限：所有账号状态为「已通过」的用户

参考截图：使用用户提供的图片screenshot.png作为客户列表页筛选区域的参考布局，筛选区域位于页面顶部，包含搜索框与多组筛选组件。

#### 4.5.1 多选筛选组件

页面顶部提供筛选区域，包含三组筛选组件，交互方式与视觉风格保持统一：

**客户分级多选筛选**：
- 筛选项：新客户、老客户
- 以标签形式展示，支持同时勾选多个分级
- 已选标签高亮显示，未选标签默认样式
- 提供「清空」按钮，一键清除已选客户分级筛选条件
- 动画效果与其他多选组件保持统一（选中/取消选中时的过渡动画一致）

**客户类型多选筛选**：
- 筛选项：央企、政府、省属、其他
- 以标签形式展示，支持同时勾选多个类型
- 已选标签高亮显示，未选标签默认样式
- 提供「清空」按钮，一键清除已选客户类型筛选条件
- 动画效果与其他多选组件保持统一

**负责人筛选**：
- 筛选项：从当前系统内已通过用户中动态加载，以用户姓名展示
- 以下拉选择器或标签形式展示，支持单选或多选负责人
- 筛选范围：客户负责人关联表中owner_type为主负责人或协助负责人的用户均纳入筛选范围
- 已选负责人高亮显示，提供「清空」按钮，一键清除已选负责人筛选条件
- 动画效果与其他多选组件保持统一

**已选筛选标签展示区**：
- 筛选区域下方展示当前已选的所有筛选条件标签（客户分级、客户类型、负责人合并展示）
- 每个已选标签提供单独删除入口（点击标签上的关闭图标移除该条件）
- 提供「清空全部」按钮，一键清除所有已选筛选条件
- 无已选条件时，已选标签展示区隐藏

**筛选逻辑**：
- 客户分级、客户类型、负责人三组筛选条件之间为「AND」关系（同时满足三组条件）
- 同一组内多个选项之间为「OR」关系（满足其中任一即可）
- 筛选结果实时更新，无需点击确认按钮
- 三组筛选条件无缝集成，共同作用于客户列表数据

#### 4.5.2 客户列表

- 展示符合当前筛选条件的客户列表
- 每条记录展示：客户名称、客户类型、客户分级、所属地区、主负责人姓名
- 列表为空时展示「暂无符合条件的客户」提示
- 支持按客户名称搜索（与多选筛选条件叠加生效）
- 点击客户记录跳转至客户详情页

### 4.6 客户详情页

页面路径：pages/customer/detail

访问权限：所有账号状态为「已通过」的用户（主负责人与协助负责人权限不同）

#### 4.6.1 基本信息展示

- 展示客户基本信息：客户名称、客户类型、客户分级、所属地区
- 展示负责人信息区域：
  - 主负责人：展示姓名，并在姓名旁显示「主负责人」权限标识
  - 协助负责人：展示姓名列表，每个姓名旁显示「协助负责人」权限标识
  - 权限标识以角标或标签形式呈现，视觉上区分主负责人与协助负责人

#### 4.6.2 编辑权限控制

- **主负责人**：可对客户基本信息进行完整编辑（客户名称、客户类型、客户分级、所属地区），编辑后保存至客户表
- **协助负责人**：只能查看客户基本信息，不展示编辑入口；可添加跟进记录
- **其他已通过用户**（非负责人）：只能查看客户基本信息，不展示编辑入口，不展示添加跟进记录入口
- 后端接口层对编辑请求进行权限校验，非主负责人的编辑请求直接拒绝

#### 4.6.3 跟进记录

- 展示该客户的所有跟进记录，按时间倒序排列
- 每条跟进记录展示：记录内容、记录人姓名、记录时间
- 主负责人与协助负责人均可添加跟进记录，填写记录内容后提交
- 跟进记录提交后不可删除、不可修改

#### 4.6.4 转移历史

- 展示该客户的所有转移历史记录，按转移时间倒序排列
- 每条转移记录展示：转出负责人姓名、转入负责人姓名、转移原因、转移时间、操作人姓名
- 转移历史只读，不提供任何编辑或删除入口
- 无转移记录时展示「暂无转移历史」提示

### 4.7 审批工作台

#### 4.7.1 用户审批列表页

页面路径：pages/admin/user-approval/index

访问权限：超级管理员、审批管理员

- 展示用户主表中所有账号状态为「待审批」的用户列表，默认按申请时间正序排列（最早申请优先）
- 每条记录展示：手机号、姓名、申请理由、申请时间
- 支持按手机号、姓名搜索
- 每条记录提供「查看详情」入口
- 待审批数量以角标形式展示在工作台入口处
- 列表为空时展示「暂无待审批记录」提示

#### 4.7.2 用户审批详情页

页面路径：pages/admin/user-approval/detail

访问权限：超级管理员、审批管理员

- 展示用户完整申请信息：手机号、姓名、申请理由、申请时间
- **审批操作区域**：
  - 「通过」操作：可选填审批备注，点击「确认通过」后：
    - 账号状态更新为「已通过」
    - 审批时间、审批人ID、审批备注写入用户主表
    - 向审批日志表写入一条记录（审批类型=用户审批，操作动作=通过）
    - 向用户发送审批通过通知（小程序订阅消息）
  - 「驳回」操作：审批备注必填（驳回原因），点击「确认驳回」后：
    - 账号状态更新为「已驳回」

    - 向审批日志表写入一条记录（审批类型=用户审批，操作动作=驳回）
    - 向用户发送审批驳回通知（小程序订阅消息，含驳回原因）
  - 驳回时审批备注为空则阻止提交，提示「请填写驳回原因」
  - 审批日志写入失败则审批操作回滚，提示「操作失败，请重试」

## 五、业务规则与逻辑

### 5.1 注册与登录规则

- 任何人均可使用手机号提交注册申请，首次注册账号状态自动置为「待审批」
- 同一手机号仅对应一个账号，重复注册时提示「该手机号已注册」，阻止创建新账号
- 已驳回用户可重新提交申请，账号状态重置为「待审批」，申请时间更新，approve_remark、approve_time、approver_id清空
- 账号状态为「待审批」的用户登录后只可访问申请进度查询页，无法进入主功能区
- 账号状态为「已驳回」的用户登录后只可访问申请进度查询页，无法进入主功能区
- 账号状态为「已通过」的用户登录后可进入主功能区

### 5.2 路由守卫规则

- 所有页面（除注册申请页外）均受路由守卫保护
- 路由守卫校验顺序：
  - 步骤1：检查登录状态，未登录跳转至注册申请页
  - 步骤2：检查账号状态，待审批/已驳回用户访问主功能区及管理后台时，跳转至申请进度查询页
  - 步骤3：检查角色权限，普通用户访问管理后台页面时后端拦截并返回权限不足提示；审批管理员访问系统管理页面时后端拦截并返回权限不足提示；非超级管理员访问客户类型分析数据中心时后端拦截并返回权限不足提示
  - 步骤4：校验通过，正常渲染页面
- 后端接口层对每次请求进行账号状态与角色权限双重校验，越权请求直接拒绝

### 5.3 审批规则

- 用户审批由超级管理员或审批管理员在审批工作台执行
- 所有审批操作（通过/驳回）均必须写入审批日志表，写入失败则审批操作回滚
- 驳回操作必须填写审批备注（驳回原因），为空则阻止提交
- 通过操作审批备注为选填
- 审批操作提交后立即生效，账号状态实时更新
- 审批操作完成后自动向被审批用户发送小程序订阅消息通知（通过/驳回结果，驳回时含驳回原因）
- 审批日志表记录不可删除、不可修改，后端接口层强制拦截对审批日志表的UPDATE和DELETE请求

### 5.4 数据关联规则

- 所有表之间通过唯一ID关联，禁止跨表直接引用非ID字段
- 审批日志表中target_id在用户审批时关联user_id
- 审批日志表中operator_id关联执行审批操作的用户user_id
- 用户角色关联表通过user_id关联用户主表
- 客户负责人关联表通过customer_id关联客户表，通过user_id关联用户主表
- 客户转移记录表通过customer_id关联客户表，通过from_user_id、to_user_id、operator_id关联用户主表

### 5.5 枚举值强制约束规则

- 账号状态仅允许：待审批、已通过、已驳回，后端接口层拒绝任何枚举值之外的状态写入
- 审批日志审批类型仅允许：用户审批、功能审批
- 审批日志操作动作仅允许：通过、驳回
- 角色仅允许：super_admin、approval_admin、normal_user
- 客户类型仅允许：央企、政府、省属、其他
- 客户分级仅允许：新客户、老客户
- 负责人类型仅允许：主负责人、协助负责人

### 5.6 客户筛选规则

- 客户分级多选、客户类型多选、负责人筛选三组条件之间为AND关系，同组内多个选项之间为OR关系
- 筛选条件变更后列表实时更新，无需手动触发查询
- 搜索关键词与多选筛选条件同时生效，取交集结果
- 清空某组筛选条件时，仅清除该组已选项，其他组筛选条件保持不变
- 「清空全部」操作同时清除客户分级、客户类型、负责人的所有已选条件及搜索关键词

### 5.7 负责人权限规则

- 每个客户有且仅有一个主负责人，协助负责人可有多个
- 主负责人拥有客户基本信息的完整编辑权限，可添加跟进记录
- 协助负责人只能查看客户基本信息，可添加跟进记录，不可编辑客户基本信息
- 后端接口层对客户编辑请求进行负责人类型校验，非主负责人的编辑请求直接拒绝
- 负责人选择器中展示权限标识（主负责人/协助负责人），便于区分

### 5.8 客户转移规则

- 客户转移操作仅超级管理员可执行
- 转移原因为必填项，为空则阻止提交
- 转移操作完成后，客户负责人关联表中原主负责人记录更新，新主负责人记录写入
- 每次转移操作均向客户转移记录表写入一条记录，记录不可删除、不可修改
- 批量转移时，每个客户分别写入独立的转移记录
- 转移历史在客户详情页可查，按转移时间倒序排列

### 5.9 客户类型分析数据更新规则

- 后端定时任务每日凌晨自动刷新统计数据，更新各分析模块的聚合结果
- 手动刷新时，后端重新执行统计查询并返回最新数据，前端更新所有图表
- 手动刷新期间展示加载状态，刷新完成后更新「最后更新时间」
- 数据下钻展示的客户列表为实时查询，不受定时任务影响
