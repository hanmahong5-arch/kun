# 孤儿用户清理系统 - 快速使用指南

## 系统概述

本系统用于自动检测和清理"孤儿"用户（Auth用户创建成功但Profile创建失败的用户），防止手机号被占用无法重新添加的问题。

## 主要功能

### 1. 自动清理（已配置）

- **执行时间**：每天凌晨2点（UTC时间）
- **执行内容**：自动检测并记录孤儿用户
- **告警阈值**：孤儿用户数量 > 2 时触发告警标志
- **无需人工干预**：系统自动运行

### 2. 手动检查和修复

#### 步骤1：访问用户管理页面

1. 登录系统
2. 进入"系统设置" → "用户管理"
3. 找到"数据一致性检查"模块

#### 步骤2：执行检查

1. 点击"立即检查"按钮
2. 等待检查完成（通常几秒钟）
3. 查看检查结果：
   - 孤儿Auth用户数量
   - 孤儿Profile记录数量
   - 不一致数据列表

#### 步骤3：修复不一致数据（如果有）

1. 点击"一键修复"按钮
2. 确认修复操作（⚠️ 此操作不可恢复）
3. 等待修复完成
4. 查看修复结果摘要

### 3. 查看清理历史

1. 在用户管理页面点击"清理历史"按钮
2. 查看历史清理记录
3. 可以按触发类型（自动/手动）、状态（成功/失败/部分成功）筛选
4. 点击记录查看详细信息

## 常见问题

### Q1: 什么是"孤儿"用户？

**A:** 当添加用户时，系统会先在auth.users表创建Auth用户，然后在profiles表创建Profile记录。如果Auth用户创建成功但Profile创建失败，就会产生"孤儿"用户。这些用户会导致后续无法使用相同手机号创建用户。

### Q2: 为什么会产生"孤儿"用户？

**A:** 可能的原因包括：
- 网络中断
- 数据库约束冲突
- 权限问题
- 代码错误

### Q3: 多久检查一次？

**A:** 
- **自动检查**：每天凌晨2点自动执行
- **手动检查**：管理员可以随时手动检查

### Q4: 如何知道有孤儿用户？

**A:** 
- 查看清理历史，关注orphan_count字段
- 如果孤儿用户数量>2，系统会设置告警标志
- 手动检查时会显示不一致数据数量

### Q5: 修复操作安全吗？

**A:** 
- 修复操作会删除孤儿数据
- 删除前会显示将要删除的数据数量
- 需要用户确认后才执行
- 所有操作都有审计日志
- ⚠️ 删除操作不可恢复，请谨慎操作

### Q6: 如何查看清理日志？

**A:** 
- 在用户管理页面点击"清理历史"按钮
- 或者直接访问：系统设置 → 清理历史

### Q7: 清理失败怎么办？

**A:** 
1. 查看清理历史中的错误信息
2. 检查网络连接
3. 重试清理操作
4. 如果问题持续，联系技术支持

## 紧急处理

### 场景1：无法添加用户（提示手机号已存在）

**处理步骤：**
1. 进入用户管理页面
2. 点击"立即检查"
3. 如果发现孤儿Auth用户，点击"一键修复"
4. 修复完成后重新添加用户

### 场景2：大量孤儿用户（>10个）

**处理步骤：**
1. 立即执行数据一致性检查
2. 查看孤儿用户列表，确认是否为异常数据
3. 执行一键修复
4. 排查用户创建流程，找出问题根源
5. 联系技术支持

## 监控建议

### 日常监控

- **每周检查一次**清理历史
- **关注告警标志**（should_alert=true的记录）
- **监控孤儿用户数量趋势**

### 异常情况

如果发现以下情况，请立即处理：
- 孤儿用户数量持续增加
- 清理操作频繁失败
- 执行时间异常长

## 技术支持

如遇到以下情况，请联系技术支持：
- 清理操作失败
- 数据一致性检查失败
- 定时任务未执行
- 其他异常情况
